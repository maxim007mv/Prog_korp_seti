02 · терминыбаза

Определения

  • Шифр — обратимое преобразование открытых данных в зашифрованные.
  • Ключ — секретная последовательность, определяющая преобразование.
  • Шифрование/Расшифрование — прямое и обратное преобразования.
  • Дешифрирование — восстановление без знания ключа (атака).
  • Криптостойкость — трудоёмкость вскрытия.

Дополнительно

  • Гамма/гаммирование — псевдослучайная последовательность для наложения на данные.
  • Имитозащита — контроль целостности и аутентичности.
  • Синхропосылка — открытые начальные параметры алгоритма.
03 · требованиясовременные

Ключевые принципы

  • Безопасность основана на секретности ключа, а не алгоритма.
  • Низкая избыточность шифртекста, малые накладные расходы.
  • Ошибки в канале не должны приводить к потере информации.

Практические критерии

  • Время/стоимость шифрования соразмерны ценности данных.
  • Единственный реалистичный способ вскрытия — полный перебор ключа.
  • Небольшие изменения ключа/открытого текста ⇒ сильные изменения шифртекста.
04 · атакимодель противника

Классические

  • Только шифртекст
  • Известный открытый текст
  • Выбранный открытый текст

Практика

  • Частотный анализ
  • Перебор ключей (bruteforce)
  • Комбинированные методы
05 · классификацияметоды

Симметричные

  • Подстановки (замены): моно-, гомофонич., полиалф., полиграмм.
  • Перестановки (транспозиции)
  • Гаммирование (поточные)
  • Комбинированные (S/P-сети)

Асимметричные

  • Обмен ключами: Диффи—Хеллман
  • Шифрование/подпись: RSA и др.
  • Гибридные схемы: RSA + симметричный ключ
06 · заменыобщее

Подстановки: идея

Каждому символу открытого текста ставится в соответствие символ шифртекста; для полиалфавитных методов — несколько алфавитов.

Моноалфавит: Yi = k1·Xi + k2 (mod N)
Полиалфавит: Yi = Xi + ki (mod N)
07 · Вижинерполиалфавит

Формула

Yi = Xi + ki (mod N)

  • Ключ — слово/фраза, повторяется по длине текста.
  • Смещает символы по алфавиту на значения ключа.

Пример

ОТКРЫТЫЙ: «ЗАМЕНА» · КЛЮЧ: «КЛЮЧ» → ШИФР: «ТМКЭШМ»

Частотный анализ усложнён по сравнению с моноалфавитной заменой.

08 · Бофортвариант

Формулы

yi = ki − xi (mod N) · либо · yi = xi − ki (mod N)

Гомофон. замена

Один символ → несколько вариантов в шифртексте для выравнивания частот.

09 · Плейферполиграммная

Шифр Плэйфера

  • Матрица алфавита 5×5, шифруются пары символов.
  • Правила: одна строка/столбец → замена соседями; прямоугольник → противоположные углы.

Пример: «ШИФР ПЛЭЙФЕРА» → «РДИЫ,-СТ-И.ЖЮБ»

10 · Вернамone-time pad

Идея

  • Бинарное представление + случайная гамма (m=2).
  • Абсолютная стойкость при уникальной, равнодлинной ключ-ленте.

Исторический контекст

Гильберт Вернам, AT&T (1926). Телеграфный код Бодо как пример кодирования символов.

11 · перестановкитранспозиции

Линейная перестановка

пример ключа: 3-8-1-5-2-7-6-4

Матрица по ключам k₁/k₂

Запись по строкам по k₁, чтение по столбцам по k₂. Усложнение анализа.

12 · особыегамильтоновы пути

Шифрование по гамильтоновым путям

Уникальный алгоритм и ключ для каждого пользователя ↔ соответствует идее одноразовой ленты: максимальная стойкость при уникальности параметров.

13 · аналитикаобщие

Матрицы

Y = C·X

Линейные преобразования над векторами символов.

Прочие

  • Задача ранца (скалярное произведение с вектором C)
  • Полиномы по mod p
  • Экспоненциальные шифры y = a^{x} (mod p)
14 · примерY=C·X

Пример кодирования

«ПРИКАЗ» → числа по алфавиту → умножение на матрицу C → вектор шифртекста.

 C = |1 3 2|
     |2 1 5|
     |3 2 1| → Y = C·X
15 · ранцевыйскаляр

Идея

y = C · si, si ∈ {0,1}ⁿ

Пример

Алфавит → 5‑битные коды → умножение на C={1,3,5,7,11} ⇒ последовательность чисел.

16 · полиномmod p

Преобразование

yi = xi³ + 2xi² + 3xi + 4 (mod 991)

Пример

«ПРИКАЗ» → числа → подстановка в полином → шифртекст: 696, 591, 922, …

17 · экспонентаa^{x} mod p

Преобразование

yi = a^{xi} (mod p), пример: a=3, p=991

Свойства

Опирается на сложность дискретного логарифмирования.

18 · потокгамма

Конечная vs бесконечная гамма

  • Фраза/семя ПСЧ vs генератор ПСЧ
  • Период > длины текста → перебор ключа становится единственным методом

ЛКМ‑генератор

Ti+1 = (A·Ti + C) mod M

19 · комбинированныеS/P-сети

От ЛЮЦИФЕР к DES

  • Чередование S‑блоков (нелинейность) и P‑блоков (перестановки).
  • Аппаратная реализация → высокая производительность.
20 · DESстандарт

Параметры

  • Блок: 64 бита
  • Ключ: 56 бит (8×7‑бит ASCII)
  • 16 раундов, режимы CBC / ECB

Идея Фейстеля

R1 = L0 ⊕ F(R0, K), L1 = R0 → обратимость при смене порядка ключей.

21 · ГОСТ 28147‑89российский

Ключ и блок

  • Ключ: 256 бит (8×32‑бит слов)
  • Блок: 64 бита → деление на A(0) и B(0)

Раундовая функция

  • Сложение [+] по mod 2³², затем S‑подстановка (8 таблиц × 4 бита)
  • Циклический сдвиг влево и обмен половин
22 · PKCасимметрия

Зачем

  • Обмен ключами без общего секрета
  • Цифровая подпись
  • Комбинация с симметричными алгоритмами

Примеры

Диффи—Хеллман, RSA и их практические режимы.

23 · DHобмен ключами

Параметры

p — простое, g — первообразный корень (mod p)

Протокол

  1. Стороны выбирают секреты a и b
  2. A = g^a mod p, B = g^b mod p — обмен
  3. K = B^a = A^b = g^{ab} mod p
24 · RSAгенерация ключей

Генерация

  1. Выбрать большие простые p, q
  2. N = p·q, φ(N) = (p−1)(q−1)
  3. Выбрать e, gcd(e, φ(N)) = 1
  4. Найти d: d·e ≡ 1 (mod φ(N))

Ключи

Открытый: (e, N) · Закрытый: (d, N)

25 · RSAиспользование

Формулы

Шифрование: C = M^e mod N

Расшифрование: M = C^d mod N

Практика

Передают случайный симметричный ключ, а полезные данные шифруют быстрым блочным/поточным алгоритмом.

26 · RSAстойкость

Требования

  • Случайные большие p и q, не близкие и не слишком разные
  • Малая gcd(p−1, q−1), обычно 2
  • Безопасные e (часто 65537)
  • Строгая защита закрытого ключа

Угроза

Факторизация N → восстановление φ(N) → вычисление d.

27 · стеганоскрытие факта

Термины

  • Контейнер, стего‑контейнер, пустой контейнер
  • Стего‑ключ
  • Сообщение

Идея

Не только зашифровать содержание, но и скрыть сам факт передачи.
